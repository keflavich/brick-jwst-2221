Catalog Pipeline Products
=========================

This page documents the main catalog products generated by:

- ``brick2221.analysis.crowdsource_catalogs_long``
- ``brick2221.analysis.merge_catalogs``
- ``brick2221.reduction.saturated_star_finding``

It focuses on what files are written and how filename tokens are constructed.


Common filename tokens
----------------------

Many outputs share tokenized suffixes:

- ``{desat}`` is ``_unsatstar`` when desaturated inputs are used, else empty.
- ``{bgsub}`` is ``_bgsub`` when background subtraction is enabled, else empty.
- ``{epsf_}`` is ``_epsf`` when ePSF building is enabled, else empty.
- ``{blur_}`` is ``_blur`` when PSF blur mode is enabled, else empty.
- ``{group}`` is ``_group`` when grouped DAOPHOT fitting is enabled, else empty.
- ``{visitid_}`` is ``_visitNNN`` for per-exposure runs.
- ``{vgroupid_}`` is ``_vgroupNNNN`` for per-exposure runs.
- ``{exposure_}`` is ``_expNNNNN`` for per-exposure runs.


crowdsource_catalogs_long
-------------------------

Module location:
``brick2221/analysis/crowdsource_catalogs_long.py``

Input image naming (selected by run mode)
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

- Mosaic mode (default):

  - ``{basepath}/{FILTER}/pipeline/jw0{proposal_id}-o{field}_t001_nircam_{pupil}-{filter}-{module}_i2d.fits``
  - Fallbacks include ``*_realigned-to-refcat.fits`` and ``*_i2d_unsatstar.fits``

- Per-exposure mode (``--each-exposure``):

  - Glob pattern:
    ``{basepath}/{FILTER}/pipeline/jw0{proposal_id}{field}{visitid}*{module}*{each_suffix}.fits``

Primary catalog outputs
~~~~~~~~~~~~~~~~~~~~~~~

Crowdsource catalogs (from ``save_crowdsource_results``):

- Star table:
  ``{basepath}/{FILTER}/{filter}_{module}{visitid_}{vgroupid_}{exposure_}{desat}{bgsub}{fpsf}{blur_}_crowdsource_{suffix}.fits``

- Sky/model cube (2 HDUs: sky-subtracted data, then model):
  ``{basepath}/{FILTER}/{filter}_{module}{visitid_}{vgroupid_}{exposure_}{desat}{bgsub}{fpsf}{blur_}_crowdsource_skymodel_{suffix}.fits``

- Optional PSF stamp:
  ``{basepath}/{FILTER}/{filter}_{module}{visitid_}{vgroupid_}{exposure_}{desat}{bgsub}{fpsf}{blur_}_crowdsource_{suffix}_psf.fits``

Typical ``suffix`` values:

- ``unweighted``
- ``nsky0`` (the currently active weighted branch)

DAOPHOT outputs (optional, when ``--daophot``)
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

DAOPHOT source tables (from ``save_photutils_results``):

- Basic fit catalog:
  ``{basepath}/{FILTER}/{filter}_{module}{visitid_}{vgroupid_}{exposure_}{desat}{bgsub}{epsf_}{blur_}{group}_daophot_basic.fits``

- Iterative fit catalog:
  ``{basepath}/{FILTER}/{filter}_{module}{visitid_}{vgroupid_}{exposure_}{desat}{bgsub}{epsf_}{blur_}{group}_daophot_iterative.fits``

DAOPHOT model/residual images:

- ``..._daophot_basic_model.fits``
- ``..._daophot_basic_residual.fits``
- ``..._daophot_iterative_model.fits``
- ``..._daophot_iterative_residual.fits``

Auxiliary outputs:

- If ``--bgsub`` is used, the input image stem gets:

  - ``*_background.fits``
  - ``*_bgsub.fits``

- If ``--epsf`` is used:

  - ``..._daophot_epsf.fits``
  - ``..._daophot_epsf.png``

- Diagnostic PNGs are written in the corresponding ``{FILTER}/pipeline/`` folder,
  with names containing ``catalog_diagnostics``.


merge_catalogs
--------------

Module location:
``brick2221/analysis/merge_catalogs.py``

There are two major merge stages: per-filter/per-module merges and global
multi-filter merged photometry tables.

Stage A: merge individual exposures per filter
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

Function: ``merge_individual_frames``

Input table glob:

- ``{basepath}/{FILTER}/{filter}_{module_or_detector}_visitNNN_vgroup*_expNNNNN{desat}{bgsub}{fitpsf}{blur_}_{method_suffix}{suffix}.fits``

  where:

  - ``method_suffix = crowdsource`` for crowdsource products
  - ``method_suffix = daophot`` for DAO-family products

Outputs in ``{basepath}/catalogs/``:

- All-columns table:
  ``{filter}_{module}_indivexp_merged{desat}{bgsub}{fitpsf}{blur_}_{method}{suffix}_allcols.fits``

- Reduced table:
  ``{filter}_{module}_indivexp_merged{desat}{bgsub}{fitpsf}{blur_}_{method}{suffix}.fits``

Stage B: merge across filters
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

Function: ``merge_catalogs`` (called by ``merge_crowdsource`` and
``merge_daophot``)

Primary outputs in ``{basepath}/catalogs/``:

- Master FITS table:
  ``{catalog_type}_{module}{_indivexp_if_set}_photometry_tables_merged{desat}{bgsub}{epsf_}{blur_}.fits``

- Master ECSV table:
  ``{catalog_type}_{module}{_indivexp_if_set}_photometry_tables_merged{desat}{bgsub}{epsf_}{blur_}.ecsv``

- Optional quality-cut version:
  ``..._qualcuts.fits``

- Additional separation-cut version:
  ``..._qualcuts_oksep2221.fits``

Saturated-star integration dependency
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

``merge_catalogs`` uses per-filter saturated-star catalogs when applying
``flag_near_saturated`` and ``replace_saturated``:

- Expected file:
  ``{basepath}/{FILTER}/pipeline/jw0{proposal}-o{obsnum}_t001_nircam_clear-{filter}-merged_i2d_satstar_catalog.fits``

This expected naming is important for downstream replacement/flagging.


saturated_star_finding
----------------------

Module location:
``brick2221/reduction/saturated_star_finding.py``

Functionality summary
~~~~~~~~~~~~~~~~~~~~~

- Detects DQ-saturated pixel islands.
- Fits PSF models to recover photometry/positions for saturated stars.
- Writes a saturated-star catalog for each processed input image.

Input globs used by ``main()``
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

- NIRCam branch:
  ``/orange/adamginsburg/jwst/{target}/{FILTER}/pipeline/*{module}*align*crf.fits``

- MIRI branch:
  ``/orange/adamginsburg/jwst/{target}/{FILTER}/pipeline/*mirimage_cal.fits``

Output written by current implementation
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

From ``remove_saturated_stars``:

- ``{input_filename_without_.fits}_satstar_catalog.fits``

Notes on naming compatibility
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

- ``merge_catalogs.py`` expects a merged-product-style saturated catalog named
  ``*_merged_i2d_satstar_catalog.fits`` (see dependency above).
- ``saturated_star_finding.py`` currently writes
  ``*_satstar_catalog.fits``.
- In practice, downstream workflows may require a rename/symlink or a wrapper
  step to align these names.
